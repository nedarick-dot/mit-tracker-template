

# 90-Day MIT Execution Operating System

## Overview
A web app for tracking Q2 2026 (April–June) Advisory Practice MITs. Humans enter daily execution notes; the system organizes them into weekly checkpoints, monthly milestones, and AI-generated leadership rollups. All logged-in users have full read/write access.

## Authentication
- Password-only login via Lovable Cloud (no email verification, no social login)
- All authenticated users have equal access to view and enter data across all departments

## Data Model (Supabase via Lovable Cloud)

### Core Tables
- **ap_mits** — The 4 AP-level MITs (title, description, VFO category, Q1 lookback score)
- **department_mits** — ~14 department-level MITs, each linked to a parent AP MIT, with owner, department (Ops/L1/L2/L3/Sales/CS/Events), hypothesis, inputs/outputs, and green/yellow/red definitions
- **monthly_milestones** — 3 per department MIT (April/May/June), auto-generated from the source doc's inputs/outputs, editable
- **weekly_checkpoints** — ~13 per quarter per department MIT, linked to the active month's milestone
- **daily_inputs** — Human-entered notes: date, department_mit_id, author, free-text update, optional blockers field
- **weekly_rollups** — AI-generated summaries per department + overall, stored weekly

### Seed Data (from PDF)
All AP MITs, department MITs, owners, hypotheses, green/yellow/red criteria, and initial monthly milestones will be pre-seeded from the uploaded document. No invented content.

## Pages & UX

### 1. Login Page
- Simple password entry screen (shared password or individual accounts — password-only auth)

### 2. Executive Dashboard (Home)
- Quarter-level view: progress bar showing weeks elapsed (week X of 13)
- 4 AP-level MIT cards with current status color (🔴🟡🟢), derived from department MIT statuses
- Overall health heatmap: departments × AP MITs grid showing status colors
- Latest weekly rollup summary (AI-generated) displayed prominently
- Quick links to each department view

### 3. Department View (`/department/:dept`)
- Shows all department MITs for that department (e.g., Ops has 5 MITs, L2 has 3, etc.)
- Each MIT card shows: title, owner, parent AP MIT link, current month milestone, status color
- Expandable to see green/yellow/red criteria from the source doc
- Timeline view showing milestones across April → May → June

### 4. MIT Detail View (`/mit/:id`)
- Full MIT detail: hypothesis, problem, inputs/outputs, green/yellow/red definitions
- **Monthly Milestones** — 3 cards for April, May, June with expected deliverables and status
- **Weekly Checkpoint Panel** — Current week highlighted, showing aggregated daily inputs for that week
- **Daily Input Form** — Text area for "What did you do today toward this MIT?", optional blockers, submit button
- **History** — Scrollable feed of all daily inputs, grouped by week
- **AI Interpretation** — Weekly rollup snippet for this specific MIT

### 5. Daily Input Page (`/daily`)
- Quick-entry view: select department → see active MITs → enter today's update for one or more MITs
- Shows what you've already entered today
- Designed for mobile-friendly fast entry

### 6. Weekly Leadership Rollup (`/rollups`)
- List of weekly rollups by date
- Each rollup contains: per-department summary, cross-department themes, blockers, overall status assessment
- Generated by AI from that week's daily inputs
- Read-only view

## AI Rollup Generation
- Edge function triggered weekly (or on-demand by any user)
- Reads all daily inputs for the past 7 days
- Groups by department and MIT
- Generates a structured summary: what progressed, what's blocked, status assessment vs. green/yellow/red criteria
- Stores result in `weekly_rollups` table and displays on dashboard + rollup page

## Monthly Milestone Logic
- April milestones are active weeks 1–4, May weeks 5–9, June weeks 10–13
- Weekly checkpoints are automatically associated with the active month
- Status of each milestone is derived from the daily inputs and AI interpretation during that month

## Visual Design
- Clean, professional dashboard aesthetic (dark sidebar navigation, light content area)
- Department color coding consistent throughout (each of the 7 departments gets a distinct color)
- Status colors: green/yellow/red matching the source doc's scoring system
- Responsive design for desktop primary, mobile for daily input

## Implementation Phases
1. **Database + Auth + Seed Data** — Set up tables, seed all MIT content from the PDF, configure auth
2. **Core Views** — Executive dashboard, department view, MIT detail with daily input form
3. **Daily Input Flow** — Quick-entry page, input history, weekly grouping
4. **AI Rollups** — Edge function for generating weekly summaries, rollup display page
5. **Polish** — Status derivation logic, milestone tracking, responsive refinements
